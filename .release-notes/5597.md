## Tighten `encode/base64` output with RFC4648

Previously it was possible to coax RFC non-compliant outputs from this package.  This change enforces the output to match the RFC4648 specifications.

`Base64.encode` Takes a single argument, the input string and provides an RFC mandated padded result.

"Implementations MUST include appropriate pad characters at the end of encoded data unless the specification referring to this document explicitly states otherwise." -- RFC4648 §3.2

`Base64.encode_url` A bug in the "base64url" encoding (see: RFC4648 §5) function resulted in nulls being embedded in the output where a padded result would have inserted the pad character.  This has been corrected.

```pony
// Before: trailing characters were NUL bytes, not stripped
Base64.encode_url("foob")   // "Zm9vYg\x00\x00"  (8 bytes)
Base64.encode_url("fooba")  // "Zm9vYmE\x00"     (8 bytes)

// After
Base64.encode_url("foob")   // "Zm9vYg"   (6 bytes)
Base64.encode_url("fooba")  // "Zm9vYmE"  (7 bytes)
Base64.encode_url("foob", true)  // "Zm9vYg==" — pass true to keep padding
```
