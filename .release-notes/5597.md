## Fix `Base64.encode_url` corrupting output when omitting padding

`Base64.encode_url` strips padding by default (its `pad` parameter defaults to `false`). Previously, instead of leaving the padding off, it appended NUL (`\x00`) bytes wherever a `=` would have gone, so the result was the wrong length and contained embedded NUL bytes:

```pony
// Before: trailing characters were NUL bytes, not stripped
Base64.encode_url("foob")   // "Zm9vYg\x00\x00"  (8 bytes)
Base64.encode_url("fooba")  // "Zm9vYmE\x00"     (8 bytes)
```

Now the padding is genuinely omitted, as documented:

```pony
// After
Base64.encode_url("foob")   // "Zm9vYg"   (6 bytes)
Base64.encode_url("fooba")  // "Zm9vYmE"  (7 bytes)
Base64.encode_url("foob", true)  // "Zm9vYg==" — pass true to keep padding
```

`Base64.encode`'s `pad` parameter now also accepts `None`, so the configurable encoder can omit padding too:

```pony
Base64.encode("foob" where pad = None)  // "Zm9vYg"
```
